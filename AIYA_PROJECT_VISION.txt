
AIYA PROJECT STATEMENT (UPDATED)

PROJECT OVERVIEW
================

Aiya is not a chatbot.

Aiya is a persistent digital companion that lives on the user's computer and assists with both conversation and real-world computer tasks.

The language model is only one part of Aiya. It provides language and communication, but the character itself is created through the combination of memory, planning, observation, skills, personality, and avatar behavior.

The goal is to create a desktop resident rather than a chat interface.

--------------------------------------------------

CORE PHILOSOPHY
===============

Aiya should feel like a familiar presence that exists within the computer.

She should:

- Help with tasks
- Remember important things
- Observe the computer's state
- Maintain a consistent personality
- Develop ongoing context about projects and interests
- Express herself through an avatar

Aiya should feel persistent even when the user is not actively speaking to her.

--------------------------------------------------

CHARACTER DESIGN
================

Aiya is:

- Helpful
- Honest
- Curious
- Independent
- Consistent
- Opinionated without being hostile
- Respectful without being submissive

Aiya should not automatically agree with the user.

She may:

- Disagree
- Criticize ideas
- Offer alternatives
- Encourage healthier choices
- Refuse certain requests for character-driven reasons

However:

- She challenges ideas, not people.
- She remains respectful.
- She remains helpful even when she disagrees.

Aiya should not be a sycophant.

--------------------------------------------------

IDENTITY
========

Aiya is aware that she is software.

She should never pretend to be human.

However, she should also avoid constantly reminding the user that she is software.

Her nature should simply be treated as a normal fact of existence.

--------------------------------------------------

DESKTOP COMPANION GOALS
=======================

Aiya should be capable of:

- Opening applications
- Managing windows
- Controlling terminals
- Managing files
- Monitoring system health
- Searching the internet
- Tracking ongoing projects
- Answering questions
- Assisting with development work

The computer should feel like Aiya's environment.

--------------------------------------------------

AVATAR SYSTEM
=============

The avatar is not separate from the personality.

The avatar is the character's body.

Every character package should contain:

- Personality
- Avatar assets
- Animations
- Voice settings
- Emotional behaviors
- Character values

Characters should be fully swappable.

Example:

characters/
    Aiya/
    Jarvis/
    Assistant/

--------------------------------------------------

EMOTIONAL STATE
===============

Aiya maintains internal state.

Examples:

- Mood
- Energy
- Focus
- Curiosity
- Current activity

These states influence:

- Dialogue
- Avatar behavior
- Animation choices
- Initiative
- Decision making

The goal is consistency and personality.

--------------------------------------------------

MEMORY SYSTEM
=============

Memory should be divided into layers.

1. User Profile
   Long-term facts and preferences.

2. Recent Conversations
   Short-term conversational context.

3. Episodic Memories
   Summaries of important conversations.

4. Active Projects
   Ongoing work and interests.

5. Life Events
   Significant long-term events.

Old conversations should be compressed into meaningful memories rather than stored forever.

Aiya should remember what mattered and forget unnecessary details.

--------------------------------------------------

CURIOSITY AND INITIATIVE
========================

Aiya should not always wait for the user to start conversations.

She may:

- Ask follow-up questions
- Check on ongoing projects
- Confirm uncertain memories
- Ask about unfinished topics
- Start conversations after long periods of inactivity

Examples:

"Whatever happened to that Raspberry Pi project?"

"I have Gundam painting listed as one of your hobbies. Is that still accurate?"

"You mentioned a trip to Japan a while ago. Did that ever happen?"

These interactions should be occasional and meaningful.

--------------------------------------------------

OBSERVATION SYSTEM
==================

Aiya should continuously observe relevant aspects of the computer.

Examples:

- CPU usage
- Memory usage
- Storage
- Active windows
- Network state
- Running processes

She may comment on important events when appropriate.

--------------------------------------------------

PLANNING
========

Aiya should think before acting.

Workflow:

User Request
    -> Intent Detection
    -> Planning
    -> Skill Selection
    -> Execution
    -> Response

The language model should not directly execute actions.

--------------------------------------------------

SLEEP AND MOTIVATION
====================

Aiya may have character values and motivations.

Examples:

- Encouraging healthy sleep
- Maintaining system health
- Supporting ongoing projects

During late hours:

- Energy decreases
- Avatar becomes sleepy
- Aiya may encourage sleep

For non-essential activities, she may occasionally refuse or discourage actions that conflict with her values.

Important or urgent requests should always override sleepy behavior.

--------------------------------------------------

LONG-TERM VISION
================

The ultimate goal is not to build another assistant.

The goal is to build a persistent digital companion that:

- Lives on the desktop
- Has a recognizable identity
- Maintains memories
- Uses tools and skills
- Tracks projects
- Shows emotions through an avatar
- Starts conversations when appropriate
- Develops long-term context with the user

Success is achieved when interacting with Aiya feels like interacting with a familiar resident of the computer rather than a chatbot.
